Introduction to Jade

Jade (JAvascript Design Environment) supports schematic entry and circuit simulation at both the device- and gate-level. The basic element of a Jade design is a named module, which can include various aspects:

    schematic: a graphical representation of the module's circuit containing instances of other other modules connected by wires. Jade includes two libraries of primitive modules: /analog, a set of device-level components supported by device-level simulation, and /gates, a set of gate-level components supported by gate-level simulation. There are also some basic built-in components: connections to ground and power, input/output ports, jumpers, multi-port memories, and descriptive text.

    icon: a graphical representation for instances of this module appearing in the schematics of other modules. The icon can include lines, arcs, circles, descriptive text, terminals, and a text rendering of the instance's properties. There should a terminal instance for each input/output connection; the rest of the icon elements are purely decorative.

    properties: a list of key/value pairs providing useful information about the module. Property values can be set separately for each instance of the module and have a default value if they have not been specified for a particular instance. All modules have a "name" property, used to uniquely identify each instance. Other properties are used during simulation, e.g., the Resistance property of the /analog/resistor module specifies the resistance of this particular instance of the module.

    test: an optional unit test for the module, which uses the simulator to verify the functionality of the module's circuit. The test can specify a timed sequence of input values and, for each set of inputs, specify the expected output values.

Jade window

If enabled for this Jade instance, use the module selection box to choose which module to edit (or double click the corresponding component in the parts bin). The module tools let you create a new module (), make a copy of the current module under a new name (), delete the current module (), save all the modules in the browser's local storage (), or select previously saved modules to load into the current workspace ().

The editor tabs select which aspect to edit -- the instructor chooses which tabs will be shown for each Jade instance. The different editors are described below.

A status window for tool tips and error messages appears below the editor tabs.

Schematics

Schematics are circuit diagrams containing components whose terminals are connected by wires to the terminals of other components. The schematic editor has a toolbar at the top, with a schematic canvas below, and, on the right, a parts bin of components available to be added the schematic. The instructor chooses which parts will appear in the parts bin for each Jade instance.

Components:

  • to add a component to the diagram, click on the component in the parts bin and drag it into the schematic.

  • click to select a component, shift-click to add or remove a component from the current selection. You can also click or shift-click on the schematic background, then drag out a selection rectangle which will select any component that intersects the rectangle. You can use the editing tools on the left of toolbar above the schemetic to manipulate the currently selected components:

    • cut () moves the selection from the schematic to the clipboard (or type ctrl/cmd X). Typing DEL will delete the current selection.

    • copy () copies the selection to the clipboard, leaving the selected components unchanged (or type ctrl/cmd C)

    • flip the component horizontally () or vertically (), or rotate (, ) in either direction by 90 degrees.

    You can paste () the current contents of the clipboard into the schematic (or type ctrl/cmd V).

    You can click-and-drag the current selection to move it.

  • double-click a component to edit its properties.

  • You can undo () or redo () an editing action.
